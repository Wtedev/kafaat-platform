# منصة كفاءات | Kafaat Platform

## نبذة مختصرة

**منصة كفاءات** نظام إداري رقمي متكامل صُمِّم خصيصاً لجهة **كفاءات**، يُغطي دورة حياة المستفيد كاملةً — من التسجيل في البرامج التدريبية والمسارات التعليمية والفرص التطوعية، مروراً بنظام الموافقات وتتبع الأداء، وصولاً إلى إصدار الشهادات الرقمية تلقائياً والتحقق منها عبر رابط عام. المنصة مبنية لتلبية الاحتياج الفعلي لكفاءات وتشغيلها كمنظومة إدارية موحدة.

---

## لماذا هذا المشروع؟

قبل بناء النظام، كانت إدارة البرامج تعتمد على:

- **متابعة يدوية** لطلبات التسجيل والموافقات عبر وسائل تقليدية
- **تشتت بيانات المستفيدين** وعدم وجود سجل موحد لحضورهم وتقدمهم
- **صعوبة تتبع** درجات الاختبارات ونسب الحضور لكل برنامج
- **إصدار الشهادات يدوياً** دون آلية تحقق موثوقة
- **غياب لوحة بيانات** تمنح الإدارة رؤية فورية عن الأنشطة والطلبات

المنصة تحوّل هذه العمليات إلى تجربة رقمية منظمة، موثوقة، وقابلة للتوسع.

---

## القيمة التي يقدمها النظام

- **تقليل العمل اليدوي** عبر أتمتة الموافقات وإصدار الشهادات
- **رفع جودة تجربة المستفيد** بواجهة واضحة لمتابعة حالة طلباته وشهاداته
- **تنظيم قرارات القبول والرفض** مع سجل كامل لكل إجراء
- **تتبع أداء البرامج** من خلال بيانات الحضور والدرجات
- **إدارة الساعات التطوعية** باعتماد رسمي مدعوم بحد ساعات قابل للضبط
- **إصدار شهادات موثوقة** برقم تسلسلي ورابط تحقق فريد
- **دعم الإدارة ببيانات أوضح** عبر لوحة إحصائيات فورية

## من يستخدم النظام؟

| الدور                     | الصلاحيات                                                          |
| ------------------------- | ------------------------------------------------------------------ |
| **مدير النظام** في كفاءات | صلاحيات كاملة — يُضيف الموظفين، يدير الأدوار، يشرف على كل شيء      |
| **موظف كفاءات**           | يُدير البرامج والفرص، يراجع الطلبات، يُحدّث الأداء، يُصدر الشهادات |
| **المستفيد**              | يُسجّل، يتابع تقدمه، يُحمّل شهاداته                                |
| **الزائر**                | يتصفح البرامج المتاحة ويتحقق من صحة أي شهادة                       |

> **ملاحظة:** كفاءات فقط هي من تُضيف موظفيها وتُدير صلاحياتهم من لوحة التحكم. لا يوجد تسجيل ذاتي للموظفين.

## رحلة العمل داخل المنصة

### رحلة المستفيد

1. **إنشاء حساب** والتحقق من البيانات الشخصية
2. **استعراض** البرامج التدريبية، المسارات التعليمية، أو الفرص التطوعية المتاحة
3. **إرسال طلب تسجيل** في النشاط المناسب
4. **انتظار الموافقة** من فريق كفاءات
5. **متابعة التقدم** — الحضور، الدرجات، الساعات المعتمدة
6. **الحصول على شهادة** تلقائياً عند استيفاء شروط الاجتياز

### رحلة موظف كفاءات

1. **إضافة البرامج** والمسارات والفرص التطوعية من لوحة التحكم
2. **مراجعة طلبات التسجيل** الواردة
3. **قبول أو رفض الطلبات** مع تسجيل السبب عند الرفض
4. **تحديث الحضور والدرجات** لكل مستفيد داخل البرامج
5. **اعتماد الساعات التطوعية** ومتابعة اكتمال الحد المطلوب
6. **إصدار الشهادات** ومتابعة طلبات التحقق منها

| **نظام الموافقات**           | قبول / رفض / إلغاء مع سجل كامل لكل قرار                           |
| **متابعة الحضور والدرجات**   | حضور بالنسبة المئوية + درجة اختبار لكل مستفيد                     |

## Technical Highlights

هذا القسم يعكس القرارات المعمارية خلف المشروع:

- **Service Layer** منفصلة عن Controllers — منطق الأعمال موزّع في `CertificateService`, `ProgramRegistrationService`, `VolunteerHoursService`
- **Polymorphic relationships** — نموذج `Certificate` واحد يخدم البرامج، المسارات، والفرص التطوعية
- **Idempotent certificate issuance** — لا يمكن إصدار شهادتين لنفس المستفيد في نفس النشاط
- **Role-based access control** — Spatie Laravel Permission مع middleware حماية لجميع المسارات
- **Admin Panel** مبني على Filament v5 مع Custom Actions وWidgets مخصصة
- **Certificate verification flow** — كود تحقق عشوائي 32 حرفاً، صفحة عامة مستقلة
- **Arabic-first interface** — RTL كامل في الواجهة الأمامية ولوحة التحكم
- **PostgreSQL** عبر Supabase مع Connection Pooling

## Setup

```bash
composer install
npm install
cp .env.example .env
php artisan key:generate
# اضبط DB_* في .env ثم:
php artisan migrate --seed
npm run build
php artisan serve
```

المتصفح: `http://localhost:8000` — لوحة التحكم: `http://localhost:8000/admin`

## Environment Notes

- **لا يُرفع ملف `.env`** — يحتوي على بيانات حساسة وهو مُستثنى من Git
- إعادة تعيين كلمة المرور تتطلب إعداد `MAIL_*` في `.env`
- للتجربة المحلية: `MAIL_MAILER=log` يُسجّل الرسائل في `storage/logs/laravel.log`
