# إعداد مفتاح البحث للهوية — `IDENTITY_LOOKUP_KEY`

## المتغير

```env
IDENTITY_LOOKUP_KEY=
```

## إنشاء مفتاح قوي

```bash
php -r "echo base64_encode(random_bytes(32));"
```

## التخزين

- Secret manager في بيئة النشر (Railway variables، Vault، …).
- **لا** commit في Git.
- **لا** طباعة في console أو logs.

## النسخ الاحتياطي

- فقدان المفتاح يمنع **التحقق من التكرار** للهويات الجديدة حتى إعادة الضبط.
- **لا** يؤثر على فك تشفير ciphertext (يعتمد `APP_KEY`).
- تغيير HMAC key **لا** يفك تشفير الهويات القديمة لكن يكسر مطابقة hash الحالية — يتطلب خطة rotation مستقبلية.

## Rotation (مستقبلاً)

1. إيقاف التسجيل مؤقتاً.
2. توليد مفتاح جديد.
3. إعادة حساب lookup_hash لكل السجلات (batch job — **غير منفذ في هذه المرحلة**).
4. تفعيل المفتاح الجديد.

## فشل التطبيق

إذا كان المفتاح غير مضبوط، عمليات الهوية ترمي `RuntimeException` واضحة.
