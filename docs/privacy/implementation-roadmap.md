## المرحلة 2 — بيانات الحساب والتسجيل

- الاسم الرباعي (أو حقول منظمة) بدل/إلى جانب `name`
- رقم الهوية (إن اعتُمد إدارياً) مع validation
- جمع/تحديث تاريخ الميلاد من البوابة (PII فقط — **بدون** فلترة عمر للبرامج)
- توحيد الجوال والتحقق من uniqueness إن لزم
- تحديث نماذج التسجيل والملف وFilament
- migrations + baseline tests محدّثة

**خارج النطاق:** SMS، تسجيل دخول بالجوال، minimum_age للبرامج.

## المرحلة 3 — سياسة الخصوصية والإقرارات

- إصدارات سياسة الخصوصية والشروط
- تسجيل موافقة عند التسجيل (version + timestamp)
- عرض سياسة محدّثة للمستخدمين الحاليين عند التغيير

## المرحلة 4 — حماية السيرة وقاعدة المرشحين الداخلية

- نقل CV/avatars إلى تخزين خاص (non-public)
- URLs موقّعة أو download عبر authorization
- سجل وصول داخلي للسيرة (موظفون)
- **بدون** مشاركة خارجية أو وحدة فرص وظيفية

## المرحلة 5 — الصلاحيات والسجلات

- Audit log لعمليات الموظفين (عرض/تصدير/تعديل PII)
- مراجعة Policies وexports
- تقليل PII في exports الافتراضية

## المرحلة 6 — مركز الخصوصية والحذف والاحتفاظ

- تصدير بيانات المستفيد
- طلب حذف الحساب (workflow + retention)
- سياسات احتفاظ sessions/logs/registrations

## المرحلة 7 — التقوية والاختبارات النهائية

- SESSION_ENCRYPT، security headers، CI
- encrypted casts حيث يلزم
- اختبارات regression كاملة + مراجعة أمنية
