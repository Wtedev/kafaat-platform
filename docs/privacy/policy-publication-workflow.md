# سير نشر سياسة الخصوصية

## الخطوات

1. **إنشاء مسودة** — Filament: إصدار، عنوان، محتوى، تاريخ سريان، `requires_reacknowledgement`.
2. **مراجعة قانونية/إدارية** — خارج النظام (لا workflow داخلي في هذه المرحلة).
3. **اعتماد المحتوى** — تأكيد النص النهائي.
4. **تحديد إعادة الإقرار** — إذا كان التعديل جوهرياً، فعّل `requires_reacknowledgement`.
5. **النشر** — إجراء «نشر» يستدعي `PrivacyPolicyPublisher` داخل transaction:
   - أرشفة الإصدار الفعّال السابق.
   - تفعيل المسودة وحساب `content_hash`.
   - مسح cache الإصدار الفعّال.
6. **إشعار المستخدمين** — غير مطبّق آلياً في هذه المرحلة (قرار إداري).
7. **متابعة الإقرارات** — Relation Manager للقراءة فقط في Filament.

## منع التعديل

- Active / Archived: عرض فقط؛ للتعديل أنشئ «إصدار جديد» من الإصدار المنشور.

## الصلاحيات

- `privacy_policy.view` — عرض.
- `privacy_policy.create` — مسودة جديدة.
- `privacy_policy.update_draft` — تعديل/حذف مسودات.
- `privacy_policy.publish` — نشر.
- `privacy_policy.archive` — (محجوز للتوسع؛ الأرشفة تلقائية عند النشر).
