# موافقة قاعدة المرشحين الداخلية

## المعنى

الموافقة تعني السماح لموظفي الجهة المخولين بدراسة الملف المهني والسيرة الذاتية وإظهاره ضمن **قاعدة المرشحين الداخلية** فقط.

## ما لا تشمله

- مشاركة السيرة مع شركة خارجية
- إرسال البيانات لطرف ثالث
- التقديم على فرصة وظيفية
- التسويق أو مشاركة مع شركاء

## الإصدارات

- جدول `candidate_pool_consent_versions` يحفظ نصوص الموافقة (draft / active / archived).
- إصدار فعّال واحد في كل وقت.
- `requires_reconsent=true` يتطلب موافقة جديدة؛ التاريخ القديم يُحفظ.

## النافذة المنبثقة

- تظهر مرة واحدة لكل إصدار فعّال بعد اكتمال الهوية.
- خياران متساويان: انضمام / عدم انضمام.
- الإغلاق يسجّل `prompted` فقط ولا يُنشئ موافقة.
- endpoint: `POST /portal/candidate-pool/prompted`.

## الحالات

| الحالة | المعنى |
|--------|--------|
| `undecided` | لم يُحدد اختيار |
| `granted` | منضم للقاعدة |
| `declined` | رفض الانضمام |
| `withdrawn` | سحب الموافقة |
| `regranted` | إعادة الانضمام بعد سحب أو إصدار جديد |

## حذف السيرة

- لا يغيّر حالة الموافقة.
- المستخدم الموافق بدون سيرة فعّالة **لا يظهر** في القائمة حتى يرفع سيرة جديدة.

## الإعدادات

مسار: `/portal/candidate-pool/settings` — عرض الحالة، الانضمام، السحب، إعادة الانضمام.
